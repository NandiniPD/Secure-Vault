🔐 Active Security Stack
AES-256

File Encryption

Encrypts file bytes → UUID.enc on disk. Unique key per file. Called in file_handler.py → aes_encrypt()

RSA-2048

Key Exchange

Locks AES key before storing in DB. Called in file_handler.py → rsa_encrypt_aes_key(). Only private key can unlock.

JWT HS256

Authentication

Signed token in every HTTP request. Called in auth.py → generate_jwt_token() after login.

SHA-256

Integrity Check

Fingerprint at upload, verified at download. Called in file_handler.py → verify_integrity()

Crypto Info
Where exactly each algorithm is used in the code
📂 Exact Code Location of Each Algorithm
AES encrypt → file_handler.py → upload_file() → enc_result = aes_encrypt(file_data, aes_key)
AES decrypt → file_handler.py → download_file() → original_data = aes_decrypt(enc_data, iv, aes_key)
RSA encrypt key → file_handler.py → upload_file() → enc_aes_key = rsa_encrypt_aes_key(aes_key, public_key)
RSA decrypt key → file_handler.py → download_file() → aes_key = rsa_decrypt_aes_key(rec["encrypted_aes_key"], private_key)
SHA-256 hash → file_handler.py → upload_file() → original_hash = compute_sha256(file_data)
SHA-256 verify → file_handler.py → download_file() → verify_integrity(original_data, rec["sha256_hash"])
JWT generate → auth.py → login_user() → token = generate_jwt_token(user_id, username)
JWT verify → run.py → @jwt_required → payload = verify_jwt_token(token)
